# Project: Sentiment Analyser for Twitter

This project benchmarks multiple NLP vectorization techniques and machine learning classifiers for sentiment classification. A total of 64 pipelines were evaluated using combinations of traditional text representations, embedding-based representations, and supervised classifiers. The goal was to identify the most effective sentiment analysis pipeline for the given dataset.

## Objective:
The objective of this project was to compare different Natural Language Processing (NLP) pipelines for sentiment classification.

Each pipeline consisted of:

1) A vectorization method to convert text into numerical features
2) A classifier to predict sentiment labels

Performance was evaluated using accuracy, macro F1-score, weighted F1-score, training time, and model size.

## Dataset:
The dataset contains text samples labeled with one of three sentiment classes:
Positive
Neutral
Negative

Two datasets were used:
80-row dataset (rapid experimentation)
300-row dataset (final evaluation)

Text data was stored in the model_text column and sentiment labels in the sentiment column.

## Methodology
### Vectorisers Evaluated
1. CountVectorizer
2. TF-IDF Word N-grams
3. TF-IDF Character N-grams
4. HashingVectorizer
5. Word2Vec Embeddings
6. FastText Embeddings
7. BERT Embeddings
8. Sentence Transformer Embeddings

### Classifiers Evaluated
1. Logistic Regression
2. Linear SVM
3. SGD Classifier
4. Multinomial Naive Bayes
5. Random Forest
6. Extra Trees
7. Gradient Boosting Trees
8. XGBoost

### Pipeline Generation
Each pipeline was generated by pairing a vectoriser with a classifier. A total of 64 pipelines were generated. Each pipeline followed the structure:
Text -> Vectoriser -> Classifier

## Implementation
The project was implemented using:

Python
Scikit-learn
XGBoost
Hugging Face Transformers
Gensim
Sentence Transformers

Custom transformer classes were developed to integrate BERT, Word2Vec, FastText, and Sentence Transformer embeddings into Scikit-learn pipelines.

## Evaluation Metrics
Accuracy
Macro F1 Score
Weighted F1 Score
Training Time
Model Size

A composite final score was calculated to rank pipeline performance while accounting for predictive performance, computational cost and storage requirements

## Results
1) Count Vectoriser Basic + Multinomial Naive Bayes: 99.9
2) Count vectoriser Basic + Logistic Regression: 99.7
3) Count vectoriser Basic + LinearSVC: 97.6

Traditional sparse representations remained highly competitive, particularly when combined with linear models. Embedding-based methods provided richer semantic representations but often required greater computational resources.

## Conclusion
This project demonstrated how different combinations of text representations and classifiers influence sentiment classification performance.

While traditional vectorization techniques such as CountVectorizer and TF-IDF produced strong baseline results, embedding-based methods offered additional semantic understanding. The benchmarking framework provides a systematic approach for evaluating NLP pipelines and can be extended to larger datasets and additional models in future work.
